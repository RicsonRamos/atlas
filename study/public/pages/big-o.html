BLOCO 01 CORE Complexidade de Algoritmos

A Ciência da Eficiência

Quando criamos software, não basta que ele funcione — ele precisa escalar. A Complexidade de Algoritmos é a ferramenta matemática para prever como tempo de execução e uso de memória crescem conforme a entrada n aumenta.

⏳

Complexidade de Tempo

Mede o número de operações que o algoritmo realiza. Responde: "Quanto mais lento fica ao processar 1 milhão de itens em vez de 10?"

💾

Complexidade de Espaço

Mede a memória RAM adicional necessária. Crítico para sistemas embarcados e pipelines de Big Data com Spark ou Pandas.

O Notação Big-O — O Pior Cenário

Big-O descreve o limite superior assintótico do tempo de execução, ignorando constantes e focando apenas no termo de maior crescimento quando n → ∞. Explore como cada classe se comporta:

O(1)
Constante
Tempo fixo independente de n. Ex: acesso a array por índice, Hash Table lookup.
O(log n)
Logarítmica
Corta o problema pela metade a cada passo. Ex: Busca Binária, B+ Tree traversal.
O(n)
Linear
Cresce proporcionalmente. Ex: varrer toda uma lista, scan completo de tabela.
O(n log n)
Linearítmica
Típico de sorts eficientes. Ex: Merge Sort, Quick Sort (caso médio), Heap Sort.
O(n²)
Quadrática
Ruim para grandes entradas. Ex: loops aninhados, Bubble Sort, comparação ingênua de pares.
O(2ⁿ)
Exponencial
Inviável acima de n≈30. Ex: força bruta em combinatória, subconjuntos de um conjunto.

📊 Análise Amortizada

O pior cenário absoluto pode ser pessimista demais. A Análise Amortizada considera o custo médio de uma operação dentro de uma sequência longa.

Exemplo Clássico: Arrays Dinâmicos (Python list, Java ArrayList).
Inserção normal custa O(1). Quando o array enche, dobra de tamanho e copia tudo: O(n).

As inúmeras operações O(1) "pagam" pelo resize ocasional. O custo amortizado por inserção permanece O(1).
